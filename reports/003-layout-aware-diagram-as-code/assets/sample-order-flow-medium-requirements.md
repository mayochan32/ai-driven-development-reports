# EC注文処理の論理的不変条件

- 注文受付後、在庫引当と不正審査を並行して開始する。
- 在庫確保と審査通過の両方が完了するまで決済へ進まない。
- 在庫不足の場合は決済を実行せず、注文失敗を返す。
- 不正審査は承認、要確認、拒否に分岐する。
- 要確認の場合は手動審査を行い、承認または拒否に分岐する。
- 審査拒否の場合は注文を中止し、引当済みの在庫があれば解放する。
- 決済失敗の場合は引当済みの在庫を解放し、注文失敗を返す。
- 決済成功後に注文DBへ確定注文を保存する。
- 注文保存失敗の場合は決済を取消または返金し、在庫を解放する。
- 注文保存成功後に注文成功をWeb画面へ返す。
- 配送依頼と確認通知は注文確定後の非同期処理として開始する。
- 配送依頼失敗は注文を取り消さず、運用担当者へ対応を依頼する。
- 通知失敗は注文を取り消さず、通知キューへ登録して再試行する。
